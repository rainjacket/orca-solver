Orca FAQ

Frequently asked questions for crossword constructors.

1. Can I use this, and how?

Yes, but currently it may not be that "out of the box" to use — especially for non-IT people. You can download an executable file from GitHub and run it from the command line, passing in a wordlist and a grid file. The command line tool only runs on a single local machine (on a configurable number of threads) and, when it completes, it simply prints out all the solutions it finds.

2. Are you planning to release the coordinator UI?

Right now the coordinator UI isn't really a hardened software application that would be intuitive for anybody to use. It's also deeply entwined into my personal setup which uses Google Cloud (requiring a Google payment account, etc.). If there's enough interest then in the future I will put some time into releasing a more fully featured system with a proper UI, instead of just the algorithm.

3. Does Orca use wordlist scoring?

Currently no, it does an exhaustive search over your entire dictionary. It wouldn't be that hard to incorporate some ways to use the wordlist scores, but I haven't had a use case for this personally because I am focusing on exhaustive search.

4. What types of grids is this good for?

Right now it's best for grids that have few total solutions. For easier-to-fill grids, it wouldn't be a very good experience to use Orca because it would start outputting an exhaustive list of billions of possible solution combinations. Orca is also currently more optimized for singular "cavern"-like regions rather than grids with multiple semi-independent regions (which you should probably still break up and fill in parts).

5. Can I incorporate this into my own software?

If you are a maker of crossword construction software, feel free to incorporate concepts from the write-up into how you think about building a solver. For more extensive integration, feel free to do so with attribution — the Rust code is open source with an MIT license.
